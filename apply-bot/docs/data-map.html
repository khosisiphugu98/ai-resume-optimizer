apply-bot — where every bit of data is stored & used

One machine, one process. The pipeline reads and writes exactly the stores below; nothing else persists. Everything is local except four outbound calls (in red), which are the only times any of your data leaves this computer.

No network exposure. The dashboard server binds to 127.0.0.1:5175 only and has no authentication — it holds an OpenAI key and can act in your name, so it is never meant to be reachable from anywhere but this machine.

SQLite — the operational database apply-bot/data/pipeline.sqlite · WAL mode · 11 tables

jobsthe board
status, tier, fit_score, reject_reason
Every posting and where it is in the pipeline. Drives all 15 board columns.
jd_text, apply_type, apply_email, external_apply_url
Fetched at enrich; read by score, tailor and apply.
resume_path, blocked_from
Pointer to the tailored PDF; the status to restore a block to.
applicationswhat was sent
channel, resume_path, submitted_at, outcome
One row per submit attempt across all three channels.
filled_json, screenshots_json, step_count
Exactly what the bot typed, for the review queue to approve.
outcome_state, outcome_at
What the employer did — the only signal calibration can use.
may hold answers
answersthe answer bank
question_norm → answer_value
Learned Q&A, normalised. An approved answer is reused so the same question never needs reviewing twice.
human_verified, source, confidence
Whether a person vouched for it; profile / human / llm_approved.
PII
eventsthe live log
ts, stage, level, message, job_id
Every line you see stream in the dashboard. Tailed over SSE so a split-process run still shows progress.
screenshot_path
Link to a step image in artifacts/.
rate_ledgerthe daily caps
linkedin_easy, external_ats, email
Per-channel counts, keyed by date. Only Easy Apply carries LinkedIn ban risk.
linkedin_pageviews, guest_fetches, audit_samples, challenges_hit
Guest fetches are unauthenticated, so counted apart from signed-in pageviews.
settingskey / value
mode
observe / review / auto.
fit_threshold, score_examples, searches_seeded
The score gate, few-shot examples built from real outcomes, and the one-time seed flag.
parked_questionswhat blocks a job
job_id, question_norm, options_json, reason
A job can block on several at once. Answering one in the dashboard releases every job waiting on it.
skill_suggestionsskills to confirm
skill_norm, job_count, status
Skills postings asked for that you haven't confirmed. Never auto-added to a resume — they queue for you to confirm or dismiss.
outboxheld email drafts
to_addr, subject, body, send_after, status
Email is the one irreversible channel — drafts sit here and auto-send unless cancelled inside the hold window.
searcheswhat to look for
keywords, location, tier, enabled
The source of truth for discovery. config.SEARCHES only seeds this once and is never read again.
blocked_companiesthe never-apply list
company_norm, reason
Checked at discovery and again at enrich. Blocking one sweeps every live job for it and cancels held drafts.
